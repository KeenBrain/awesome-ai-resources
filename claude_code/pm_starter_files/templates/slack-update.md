# Slack Update Template

For quick updates in #product-updates or similar channels. Keep it scannable.

---

## Format

```
:emoji: **[Short Title]**

[1-2 sentence summary]

**Key points:**
• Point 1
• Point 2
• Point 3

:link: [Link to full doc/spec if needed]
:thread: Questions? Reply in thread
```

---

## Example: Feature Launch

```
:rocket: **Spec Templates are live!**

You can now create and save spec templates for your team. No more copy-pasting from old docs.

**What's new:**
• Create templates with required sections
• Share templates across your workspace
• Set a default template for new specs

:link: Full details in the changelog
:thread: Feedback? Reply in thread!
```

---

## Example: Research Insight

```
:mag: **Research drop: Mid-market churn analysis**

Completed 20 interviews with churned mid-market customers. TL;DR: multi-product support is the #1 gap.

**Top reasons for churn:**
• Can't manage multiple product lines (65%)
• Performance issues at scale (45%)
• Missing enterprise permissions (40%)

:link: Full report in Research folder
:thread: Want to discuss? Let's chat
```

## Example: Status Update

```
:traffic_light: **AI Features Q1 Status: Yellow**

We're behind on the roadmap prioritization feature. Shipping 2 weeks later than planned.

**What's on track:**
• Spec drafting v2 (shipped last week)
• Feedback summarization (on track)

**What's behind:**
• Roadmap prioritization (delayed - API issues)

:link: Full status in weekly report
:calendar: Sync on Monday to discuss
```

---

## Tips

- Keep it under 200 words
- Use bullet points, not paragraphs
- Always include a link for "more details"
- Always invite questions in thread
- Post at reasonable times (not 11pm)
- Use @channel sparingly - only for truly important updates
