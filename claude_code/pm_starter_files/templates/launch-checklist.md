# Launch Checklist Template

Use this checklist for any significant feature launch. Not everything applies to every launch - mark N/A as appropriate.

## Pre-Launch (2+ weeks out)

### Product
- [ ] PRD approved by stakeholders
- [ ] Success metrics defined and baseline measured
- [ ] Feature flagged for controlled rollout
- [ ] QA complete - no P1/P2 bugs
- [ ] Accessibility audit passed
- [ ] Performance testing complete

### Design
- [ ] Final designs approved
- [ ] Design QA against implementation
- [ ] Dark mode verified (if applicable)
- [ ] Mobile responsive verified (if applicable)
- [ ] Empty states designed

### Engineering
- [ ] Code complete and merged
- [ ] Feature flag configured
- [ ] Monitoring/alerting set up
- [ ] Runbook for incidents created
- [ ] Database migrations tested
- [ ] Rollback plan documented

### Analytics
- [ ] Tracking events implemented
- [ ] Dashboard created or updated
- [ ] A/B test configured (if applicable)

---

## Launch Week

### Documentation
- [ ] Help center article written
- [ ] In-app tooltips/guides configured
- [ ] API documentation updated (if applicable)
- [ ] Changelog entry drafted

### Customer Success
- [ ] CS team briefed on feature
- [ ] FAQ document created
- [ ] Known issues/limitations documented
- [ ] Support escalation path defined

### Marketing (for larger launches)
- [ ] Blog post drafted
- [ ] Email announcement drafted
- [ ] Social media posts drafted
- [ ] Website updates ready
- [ ] Sales enablement materials ready

### Communications
- [ ] Internal announcement ready (Slack)
- [ ] Customer announcement ready (if needed)
- [ ] Press/analyst briefing (if major launch)

---

## Launch Day

### Morning
- [ ] Feature flag enabled for beta users (if phased)
- [ ] Monitor error rates and performance
- [ ] CS team standing by

### Rollout
- [ ] Beta → 10% → 50% → 100% (or your rollout plan)
- [ ] Check dashboards at each stage
- [ ] Gather early user feedback

### Communications
- [ ] Internal Slack announcement sent
- [ ] Changelog published
- [ ] Customer communications sent (if applicable)
- [ ] Marketing content published (if applicable)

---

## Post-Launch (Week 1)

### Monitoring
- [ ] Daily check on error rates
- [ ] Daily check on key metrics
- [ ] Support ticket volume normal
- [ ] No critical bugs reported

### Feedback
- [ ] User feedback collected
- [ ] CS feedback collected
- [ ] Sales feedback collected
- [ ] Bugs triaged and prioritized

### Documentation
- [ ] Known issues documented
- [ ] Help content updated based on feedback

---

## Post-Launch (Week 2-4)

### Analysis
- [ ] Success metrics measured against targets
- [ ] User feedback synthesized
- [ ] Fast follows identified
- [ ] Retro scheduled

### Reporting
- [ ] Launch summary shared with stakeholders
- [ ] Learnings documented
- [ ] Case study candidate identified (if successful)
